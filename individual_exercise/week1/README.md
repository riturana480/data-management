### About the dataset
Each row represents one hotel booking, including both bookings that effectively arrived and bookings that were canceled. All hotel- and customer-identifying fields were removed by the original authors. **Four columns — `name`, `email`, `phone-number`, `credit_card` — were artificially generated by the Kaggle uploader and are NOT real data.**

## Metadata Elements Covered

Both files describe the same dataset and include the core metadata properties taught in class:

- **Identifier** — the Kaggle dataset URL is used as the persistent identifier (the Kaggle upload does not have a DOI)
- **Title** and subtitle
- **Creators** — the three original authors (Antonio, Almeida, Nunes) with affiliation
- **Contributor** — the Kaggle uploader (Mojtaba) as data curator
- **Publisher** — Kaggle (the platform hosting this redistribution)
- **Publication year** — 2020
- **Resource type** — Dataset (tabular, CSV)
- **Subjects / Keywords** — from the Kaggle tags and the subject domain
- **Dates** — collected, issued, and reservation-status date range
- **Language** — English
- **Size** (119,390 × 36; 25.11 MB) and **format** (CSV)
- **Version** — 1 (as listed on Kaggle)
- **Rights / License** — "Data files © Original Authors" as stated on Kaggle
- **Descriptions** — abstract, methods, technical info, and Kaggle-specific metadata
- **Geographic coverage** — Portugal
- **Temporal coverage** — 2015-07-01 / 2017-08-31
- **Variables measured** (schema.org only, `variableMeasured`) — all 36 columns with descriptions and the synthetic fields clearly flagged
- **Distribution** (schema.org only) — file name, format, size, URL
- **Citation** (schema.org only) — structured reference to the original *Data in Brief* paper

## A note about the identifier
The Kaggle upload itself does not have a DOI. For the primary `identifier` we therefore use the **Kaggle dataset URL** (`identifierType="URL"` in the DataCite record), which is the most stable, publicly resolvable reference for this specific version of the data.

## How to Validate

**DataCite XML** — follows [DataCite Metadata Schema 4.4](https://schema.datacite.org/meta/kernel-4.4/):
```bash
xmllint --noout --schema https://schema.datacite.org/meta/kernel-4.4/metadata.xsd hotelbooking_datacite.xml
```
Or use the online validator at <https://schema.datacite.org/>.

**schema.org JSON-LD** — can be validated at:
- [schema.org Validator](https://validator.schema.org/)
- [Google's Rich Results Test](https://search.google.com/test/rich-results)
- [JSON-LD Playground](https://json-ld.org/playground/)
